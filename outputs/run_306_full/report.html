5. Key Findings

8 Findings from 306-Request Full-Scale Study

Statistically Robust
1. Agent reasoning adds +2.61% recall@100 (+18.2% relative)
263 evaluated requests. The agent's signal-adaptive routing — adjusting strategy based on similarity_gap, route reliability, cluster engagement — consistently outperforms fixed-weight RRF. NDCG@10 improvement (+30.6%) shows the agent places the best ads first.
2. PM truncation is the #1 bottleneck (64.1% positive loss)
306 requests. Only 35.9% of positives survive PM, 2.2% reach AF. The budget constraint (500 from ~1900) is structural. FR bypass helps (64 ads skip PM) but most positives still compete for limited PM slots.
3. Three routes find 20.1 unique positives per request
PSelect: 9.5, Prod Model: 6.3, FR: 4.2. All routes have <14% overlap, confirming the multi-route architecture is the foundation of recall. No single route achieves comparable coverage alone.
4. similar_ads is the best exploration route (+3.8% recall)
Expanding from known good ads to find similar ones adds the most value. anti_negative and cluster_explorer each add ~2.3%. HSNN as standalone adds only 0.7%, confirming it should be a prod model mode.
5. pCTR (prod_prediction) is harmful for PM scoring
0.97x separation — negatives score higher. PM survival: cosine 48.6% vs pCTR 29.7%. Now using RAA pm_total_value (36.6x separation where per-request data available) with cosine fallback.
6. RAA consideration data: 63% eCPM coverage (vs 4% before)
fct_raa_consideration_data has real pm_stage_struct.total_value + is_forced_retrieval_ads. Prod model unique positives: 3.5 (pCTR) → 6.3 (eCPM) — 80% improvement in route value.
7. ML Reducer outperforms heuristic by +4.8%
ML Reducer preserves 73.3% of positives vs heuristic 68.5% at 50% reduction. Confirmed across 100-request (9.4%) and 306-request (4.8%) studies. Validates ML-driven truncation in the PM stage.
8. Next step: user-specific eCPM + combined PM scoring
Current eCPM is ad-level aggregate (not user-specific). Combining cosine (user-specific) + eCPM (ad quality) + cluster engagement (contextual) would create the strongest PM scorer. Also: widen PM budget from 500 to 700+ for mechanical recall improvement.
