4. Key Findings (306 requests, statistically robust)

8 Findings with Evidence

Full-Scale Validation
1. Agent reasoning adds +2.61% recall@100 over RRF baseline (+18.2% relative)
263 evaluated requests. Agent: 16.99%, Baseline: 14.38%. The largest improvement is in NDCG@10 (+11.05%), showing the agent excels at ranking the most relevant ads at the top. This is the clearest proof of LLM reasoning value in retrieval.
2. PM truncation is the #1 bottleneck (58.6% positive loss)
306 requests. Only 41.5% of positives survive PM, and only 2.7% reach AF. This is the single largest source of recall loss. eCPM + FR bypass improves survival but the fundamental budget constraint (500 ads from ~1900 pool) makes this structurally hard.
3. Three routes are highly complementary (<10% overlap each)
PSelect: 9.9 unique positives, FR: 9.2, Prod Model: 6.5. All three routes contribute independent signal with <10% overlap. Combined they find 25.6 unique positives per request — no single route can match this.
4. similar_ads is the best exploration route (+2.4% recall)
Confirmed at 306-request scale. similar_ads_lookup expands from known good ads to find unique positives. HSNN as exploration slightly hurts (-0.2%), confirming it should remain a prod model mode, not a standalone route.
5. ML Reducer outperforms heuristic by +7.0%
ML Reducer preserves 75.5% of positives vs heuristic 68.5% at 50% reduction. Consistent across all run sizes (100 req: +9.4%, +5.6%; 306 req: +7.0%). Validates ML-driven truncation.
6. pCTR (prod_prediction) is harmful for PM scoring
0.97x separation ratio — negatives score higher than positives. PM survival: cosine 48.6% vs pCTR 29.7%. The extraction script now uses RAA pm_total_value (36.6x separation) and median_ecpm (2.48x) instead.
7. RAA consideration data provides 63% eCPM coverage
fct_raa_consideration_data has real PM/AI/AF total_value per ad. Coverage jumped from 4% (old float_features[5166]) to 63%. Also provides real is_forced_retrieval_ads flags (82 FR ads/request avg).
8. FR bypass preserves production behavior
With RAA FR flags, 82 ads per request bypass PM truncation (matching production). FR contributes 9.2 unique positives per request. Without bypass, these ads would be subject to PM scoring and 60%+ would be lost.
