2. What Changed in the Prompt

V1 Prompt (Old)

Priority Merge

Strategy: Qualitative reasoning

1. Check similarity_gap thresholds

2. Assess route reliability (centroid_gap vs user_emb_gap)

3. Look for consensus ads

4. Check cluster engagement rates

5. Merge: consensus first, FR unique, cluster ads, prod_model, pselect to fill

Problem: Agents interpreted "merge" differently — 80% just dumped FR first, then filled. No systematic scoring. No diversity constraints.

V2 Prompt (New)

Weighted Rank Fusion + Cluster Diversity

Strategy: Algorithmic scoring with explicit weights

1. Signal regime: STRONG / MODERATE / WEAK

2. Score every ad: score = Σ weight/rank across all routes

3. Explicit weight table by regime (prod_model=3.0, FR=2.0-3.0, etc.)

4. Consensus bonus: +1.0 for 3+ routes, +2.0 for known positives

5. Cluster diversity: 10+ from high-engagement clusters, cap 40

6. Output 200 ads (vs 150 before)

Why it works: Deterministic algorithm leaves less room for agent interpretation errors. Explicit weights prevent FR-only dumping. Cluster caps ensure diversity.
